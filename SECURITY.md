# セキュリティポリシー

## 報告窓口

本CLIおよび関連サービスに関する脆弱性を発見した場合は、GitHubの Issue ではなく、
以下のいずれかの方法で運営(Realize Inc.)まで直接ご連絡ください。

- リポジトリの GitHub Security Advisories 機能から Private report を作成
- 公開が不適切な内容(認証情報の漏えい可能性、ペアリングの悪用手法等)は、
  詳細を伏せた上で Security Advisories 経由の連絡を推奨します。

## 対応方針

- 報告受領後、可能な限り速やかに内容を確認します。
- 深刻度に応じて修正版のリリース・注意喚起を行います。
- 本CLIは**ユーザーのローカル環境で動作し、credentials系ファイルには一切アクセスしない**
  設計です。認証情報(OAuthトークン等)に触れる変更提案は原則として受け入れません。
- ネットワーク通信を行うコードは `packages/cli/src/api.ts` の1ファイルに限定しています。
  この境界を壊す変更(他ファイルへの `fetch` 等の追加)は、テスト(ネットワーク遮断テスト)
  によりCIで検出されます。

## スコープ

- `packages/core`: ログ探索・パース・集計ロジック。ネットワークコード不在。
- `packages/schema`: 送信JSONのzodスキーマ(公開契約)。
- `packages/cli`: CLI本体。UI・確認プロンプト・送信処理。

サーバー側(`shibaita/web`、非公開リポジトリ)に関する報告も歓迎しますが、
本リポジトリのSECURITY.mdはCLI(公開コード)を対象範囲としています。

## 既知の設計上の限界

詳細は [THREAT_MODEL.md](./THREAT_MODEL.md) を参照してください。ローカルログの改ざんや
偽データ生成については、本サービスの性質上(自己申告値によるネタ的なランキング)構造的に
は防いでいません。これは既知の設計判断です。
