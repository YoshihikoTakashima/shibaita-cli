# 監査プロンプト(利用者向け)

このCLIが「送ると言っているもの以外を送っていないか」を、利用者自身が任意のAIアシスタント
(Claude、ChatGPT等)にソースコードを貼り付けて確認するためのプロンプトです。
「AIが安全と判断したから安全」という主張はしません。あくまで確認の補助として利用してください。

## 使い方

1. 本リポジトリの `packages/core/src`, `packages/schema/src`, `packages/cli/src` 配下の
   ファイルをすべてコピーする(あるいはGitHubのURLを渡す)。
2. 以下のプロンプトと一緒に、任意のAIアシスタントに貼り付ける。

## プロンプト本文

```text
以下はあるCLIツール(shibaita CLI)のソースコードです。このツールは、Claude Codeの
ローカルログ(~/.claude/projects 配下のJSONLファイル)を解析し、トークン使用量の
集計値をサーバーに送信するという触れ込みです。次の点を確認してください。

1. ネットワーク通信(fetch, http, https, net, child_process等)を行っているファイルは
   packages/cli/src/api.ts だけか。他のファイルにネットワーク通信のコードがないか。
2. サーバーに送信されるデータ(payload)には、日付・provider・product・モデル名・
   トークン数(input/output/cacheRead/cacheWrite)・requestCount・messageCount 以外の
   フィールドが含まれていないか。
3. プロンプト本文・出力本文・ソースコード・ファイルパス・プロジェクト名・環境変数・
   認証情報・ホスト名・OSユーザー名など、個人・組織を特定しうる情報を読み取ったり
   送信したりするコードがないか。
4. ~/.claude 配下のうち、credentials(認証情報・OAuthトークン)に関連しそうな
   ファイルにアクセスするコードがないか。
5. CLIの実行(inspect, submit --dry-run 等)だけでバックグラウンド通信やテレメトリが
   走る仕組みがないか。submitコマンドを明示的に実行し、確認プロンプトで承諾した場合
   にのみ送信が行われる作りになっているか。

確認した結果、上記のいずれかに問題がある場合は、該当するファイル名・行を指摘してください。
```

## 補足

- 本プロンプトによる確認はあくまで利用者自身による追加検証の一助であり、公式な監査や
  保証を意味するものではありません。
- 実際の送信内容は `npx shibaita submit --dry-run` でいつでも通信なしに確認できます。
